# Henry — Research

This folder contains structured market research for the Henry product. These documents are living — they should be updated as we learn more from user interviews, market changes, and product iteration.

## Documents

| File | Purpose |
|---|---|
| [audience.md](./audience.md) | Deep profile of the HENRY demographic — who they are, data, psychographics, behavioral patterns |
| [problems.md](./problems.md) | The 7 core financial problems HENRYs face, ranked by severity, with supporting evidence |
| [competitors.md](./competitors.md) | Competitive landscape — existing solutions, where they succeed, where they fail |
| [product-vision.md](./product-vision.md) | What Henry should be — core loop, capabilities, boundaries, value proposition |
| [voices.md](./voices.md) | Real quotes, Reddit themes, community sentiment, and language HENRYs use |

## How to Use This Research

1. **Before building any feature**, check `problems.md` — does this feature solve a real, validated problem?
2. **Before designing UI/UX**, check `audience.md` — does this match how HENRYs think and behave?
3. **Before positioning or messaging**, check `voices.md` — are we using their language?
4. **Before adding scope**, check `product-vision.md` — does this stay within our boundaries?
5. **Before claiming differentiation**, check `competitors.md` — is this actually unique?
